# Review Calendar — When to Check What

## Daily (if active on a project)

- [ ] Write learning files from the day's sessions (if any)
- [ ] Check `uncertainty-log.md` for unanswered questions

## Weekly

- [ ] Consolidate week's learnings → check if any pattern appears 3+ times
- [ ] If pattern confirmed → update `pattern-library.md`
- [ ] Review `improvement-suggestions.md` — pick one to implement
- [ ] Check if any calibration needs adjustment

## Monthly

- [ ] Full review of `calibrations.md` — are scoring weights still accurate?
- [ ] Review `pattern-library.md` — prune outdated patterns, add emerging ones
- [ ] Check if any new AI crawler bots need to be added (e.g., robots.txt checks)
- [ ] Review reference files for any missing content
- [ ] Check industry news: Google updates, AI search changes

## Quarterly

- [ ] Major calibration review — adjust all weights if needed
- [ ] Architecture review — does the skill structure still make sense?
- [ ] Prune old learning files (archive learnings/ older than 6 months)
- [ ] Add new command types if needed
- [ ] Validate CORE-EEAT items: any new criteria from Google or AI engines?

## Annual

- [ ] Full rewrite of skill if SEO landscape changed significantly
- [ ] Review all reference files for freshness
- [ ] Check compatibility with new agent harnesses
- [ ] Archive old template patterns

## Triggers for Immediate Review

| Event | Action |
|-------|--------|
| Google core update announced | Review CORE-EEAT items for relevance |
| New AI search engine launches | Add to geo checklist + robots.txt checks |
| Pattern fires 5+ times in month | Add to pattern-library.md with HIGH weight |
| User gets penalized | Log penalty details to learnings/, create new pattern |
| Ranking drops >30% post-advice | Review advice given, fix calibration |
