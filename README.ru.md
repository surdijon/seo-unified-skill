## Что он делает

Одна команда **аудитирует и оптимизирует статические сайты** (Astro, HTML, Hugo, 11ty) как для Google, так и для поисковых систем с ИИ. Он находит проблемы, оценивает контент по 70 критериям EEAT, проверяет готовность к цитированию ИИ и генерирует план действий — всё из одного URL.

Скилл адаптируется к **зрелости сайта** (Новый / Растущий / Устоявшийся / Доминирующий), оценивая 2-страничный локальный сайт иначе, чем 100-страничный авторитетный домен.

## Ключевые особенности

| Особенность | Значение |
|-------------|----------|
| **25+ команд** | `auto` для полного пайплайна или точечно: `head`, `speed`, `schema`, `geo`, `local`, `links`, `keywords`, `brief`, `fresh`, `recover` |
| **6-фазный авто-пайплайн** | Discovery → Content → Local → GEO → Links → Отчёт. Запускает сабагентов, корректно обрабатывает ошибки. |
| **70 критериев EEAT** | Контент, Экспертность, Репутация, Оригинальность, Техника — оценка с адаптацией по зрелости |
| **Определение зрелости** | Новый (<6м/10с) vs Растущий (6-18м/10-30с) vs Устоявшийся (18м+/30+с) vs Доминирующий (3г+/100+с). Оценка подстраивается автоматически. |
| **GEO-first** | Проверяет готовность к цитированию в ChatGPT, Perplexity, Claude, Gemini, Google AIO |
| **Условия вето** | 9 блокировщиков проверяются до оценки (блокировка robots.txt, отсутствие HTTPS, ручные действия и т.д.) |
| **Протокол истины** | Никогда не ставит "Пройдено" без видимых доказательств. Никогда не угадывает обратные ссылки. Никогда не объявляет схему валидной без проверки. |
| **Самообучение** | Каждая сессия записывает находки. Паттерны накапливаются в `evolution/pattern-library.md`. Калибровки корректируются со временем. |
| **Защита от раздувания** | Файлы обучения создаются только при неочевидных находках. Лимит в 50 файлов с архивацией. |
| **Аудит head-тегов** | 40 пунктов в 6 категориях (meta, OG, schema, иконки, geo, соцсети) с приоритетами P0-P3 |
| **Структурированный вывод** | Каждая команда выдаёт: оценку, резюме, находки, план действий с оценкой усилий |
| **Кроссплатформенность** | Работает с Claude Code, Open Code, Cursor, Cline, Aider, Codex CLI, Gemini CLI |

## Команды

### Старт
| Команда | Что делает |
|---------|------------|
| `help` | Показывает все команды с примерами |
| `auto <url>` | Полный пайплайн: 6 фаз, сабагенты, составная оценка |
| `audit <url>` | 3-слойный аудит + оценка EEAT + план действий WaaS |
| `head <url>` | Аудит 40 head-тегов (meta, OG, schema, иконки, geo) |

### Контент
| Команда | Что делает |
|---------|------------|
| `keywords <тема>` | Приоритезированные ключевые слова с картой интентов |
| `brief <тема>` | Готовый редакторский бриф + анализ SERP + структура |
| `content <url>` | Полный бенчмарк EEAT из 70 критериев + план улучшений |
| `clusters <тема>` | Карта тематического кластера + столб/подтемы + перелинковка |
| `fresh <url>` | Аудит свежести контента (устаревшие статистики, даты, примеры) |
| `recover <url>` | Триаж угасающего контента (обновить / объединить / перенаправить) |

### Техника
| Команда | Что делает |
|---------|------------|
| `speed <url>` | Анализ Core Web Vitals (LCP, CLS, INP) + корневые причины |
| `technical <url>` | Диагностика Краулинг → Индекс → Сигналы |
| `schema <url>` | Обнаружение, валидация и генерация JSON-LD |

### Локальное
| Команда | Что делает |
|---------|------------|
| `local <бизнес>` | Аудит Google Business Profile + NAP + отзывы |
| `napcheck <бизнес>` | Согласованность NAP в каталогах |
| `links <url>` | Анализ профиля обратных ссылок + кампания линкбилдинга |
| `gap <бизнес>` | Конкурентный разрыв: что есть у топ-3, чего нет у вас |
| `serp <ключевое_слово>` | Продвинутый сниппет + PAA + нацеливание на богатые результаты |

### ИИ / GEO
| Команда | Что делает |
|---------|------------|
| `geo <url>` | Аудит видимости в ИИ-поисковиках + проверка llms.txt |
| `citecheck <url>` | Проверка, цитируют ли ИИ-движки сайт |

### Эволюция
| Команда | Что делает |
|---------|------------|
| `memory` | Показать накопленные знания и паттерны |
| `reset` | Архивировать текущие знания и начать заново |

## Быстрый пример

```bash
auto https://santexnik-dijon.fr
```

Скилл:
1. Проверит, могут ли ИИ-краулеры доступаться к сайту (robots.txt)
2. Проанализирует head-теги, скорость, схему
3. Оценит контент по 70 критериям EEAT (адаптировано под зрелость)
4. Проверит Google Business Profile
5. Проверит, цитируют ли ChatGPT / Perplexity этот сайт
6. Сравнит с 3 основными конкурентами

→ **Полный отчёт с приоритезированными действиями, оценкой усилий и готовым к копированию кодом.**

## Система самоэволюции

Скилл улучшается со временем **без настройки**:

1. После каждой команды находки записываются в `evolution/learnings/`
2. Повторяющиеся паттерны накапливаются в `evolution/pattern-library.md`
3. Корректировки оценки отслеживаются в `evolution/calibrations.md`
4. При превышении 50 файлов старые обучения архивируются и консолидируются
5. Файлы обучения создаются только при неочевидных находках

Файлы ссылок НИКОГДА не изменяются автоматически. Пользователь решает о калибровках.
