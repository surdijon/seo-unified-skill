## ما يفعله

أمر واحد **يدقق ويحسن المواقع الثابتة** (Astro, HTML, Hugo, 11ty) لكل من Google ومحركات البحث بالذكاء الاصطناعي. يكتشف المشاكل، ويقيم المحتوى وفق 70 معيار EEAT، ويتحقق من جاهزية الاقتباس في الذكاء الاصطناعي، ويولد خطة عمل — كل ذلك من رابط واحد.

تتكيف المهارة مع **نضج الموقع** (جديد / نامٍ / راسخ / مهيمن)، وتقييم موقع محلي من صفحتين بشكل مختلف عن نطاق سلطة من 100 صفحة.

## الميزات الرئيسية

| الميزة | المعنى |
|--------|--------|
| **25+ أمراً** | `auto` للخط الكامل، أو أوامر محددة: `head`, `speed`, `schema`, `geo`, `local`, `links`, `keywords`, `brief`, `fresh`, `recover` |
| **خط أنابيب تلقائي بـ 6 مراحل** | Discovery → Content → Local → GEO → Links → تقرير. يشغل وكلاء فرعيين، يعالج الأخطاء بشكل أنيق. |
| **70 معيار EEAT** | المحتوى، الخبرة، السمعة، الأصالة، التقنية — تقييم متكيف مع النضج |
| **كشف النضج** | جديد (<6أش/10ص) vs نامٍ (6-18أش/10-30ص) vs راسخ (18أش+/30+ص) vs مهيمن (3س+/100+ص). التقييم يتكيف تلقائياً. |
| **GEO أولاً** | يتحقق من جاهزية الاقتباس في ChatGPT, Perplexity, Claude, Gemini, Google AIO |
| **شروط الاعتراض** | 9 عوائق تُفحص قبل التقييم (حظر robots.txt، عدم وجود HTTPS، إجراء يدوي، إلخ) |
| **بروتوكول الحقيقة** | لا يحدد أبداً "ناجح" دون دليل مرئي. لا يخمن أبداً الروابط الخلفية. لا يعلن أبداً صحة المخطط دون التحقق. |
| **تعلم ذاتي** | كل جلسة تسجل النتائج. تتراكم الأنماط في `evolution/pattern-library.md`. تتكيف المعايرة مع الوقت. |
| **حماية من التضخم** | ملفات التعلم تُنشأ فقط عند وجود نتائج غير واضحة. حد 50 ملفاً مع الأرشفة. |
| **تدقيق وسوم head** | 40 بنداً في 6 فئات (meta, OG, schema, أيقونات, geo, تواصل اجتماعي) بمستويات أولوية P0-P3 |
| **مخرجات منظمة** | كل أمر يقدم: تقييماً، ملخصاً تنفيذياً، نتائج، خطة عمل مع تقدير الجهد |
| **متعدد المنصات** | يعمل مع Claude Code, Open Code, Cursor, Cline, Aider, Codex CLI, Gemini CLI |

## الأوامر

### البداية
| الأمر | الوظيفة |
|-------|---------|
| `help` | عرض جميع الأوامر مع أمثلة |
| `auto <url>` | خط أنابيب كامل: 6 مراحل، وكلاء فرعيون، تقييم مركب |
| `audit <url>` | تدقيق 3 طبقات + تقييم EEAT + خطة عمل WaaS |
| `head <url>` | تدقيق 40 وسماً من وسوم head (meta, OG, schema, أيقونات, geo) |

### المحتوى
| الأمر | الوظيفة |
|-------|---------|
| `keywords <موضوع>` | كلمات مفتاحية ذات أولوية مع خريطة النية |
| `brief <موضوع>` | موجز تحريري جاهز + تحليل SERP + مخطط |
| `content <url>` | معيار EEAT كامل من 70 معياراً + خطة تحسين |
| `clusters <موضوع>` | خريطة عنقود موضوعي + عمود/مواضيع فرعية + ترابط داخلي |
| `fresh <url>` | تدقيق نضارة المحتوى (إحصائيات وتواريخ وأمثلة قديمة) |
| `recover <url>` | فرز تراجع المحتوى (تحديث / دمج / إعادة توجيه) |

### التقني
| الأمر | الوظيفة |
|-------|---------|
| `speed <url>` | تحليل Core Web Vitals (LCP, CLS, INP) + الأسباب الجذرية |
| `technical <url>` | تشخيص الزحف → الفهرس → الإشارات |
| `schema <url>` | كشف والتحقق من صحة وتوليد JSON-LD |

### المحلي
| الأمر | الوظيفة |
|-------|---------|
| `local <نشاط_تجاري>` | تدقيق Google Business Profile + NAP + تقييمات |
| `napcheck <نشاط_تجاري>` | اتساق NAP عبر الأدلة |
| `links <url>` | تحليل ملف الروابط الخلفية + حملة بناء الروابط |
| `gap <نشاط_تجاري>` | الفجوة التنافسية: ما يمتلكه أفضل 3 منافسين وما ينقصك |
| `serp <كلمة_مفتاحية>` | المقتطف المميز + PAA + استهداف النتائج الغنية |

### الذكاء الاصطناعي / GEO
| الأمر | الوظيفة |
|-------|---------|
| `geo <url>` | تدقيق الرؤية في محركات البحث بالذكاء الاصطناعي + التحقق من llms.txt |
| `citecheck <url>` | التحقق مما إذا كانت محركات الذكاء الاصطناعي تستشهد بالموقع |

### التطور
| الأمر | الوظيفة |
|-------|---------|
| `memory` | عرض التعلم المتراكم والأنماط |
| `reset` | أرشفة التعلم الحالي والبدء من جديد |

## مثال سريع

```bash
auto https://سباك-ديجون.fr
```

ستقوم المهارة بـ:
1. التحقق من قدرة زاحفي الذكاء الاصطناعي على الوصول إلى الموقع (robots.txt)
2. تحليل وسوم head والسرعة والمخطط
3. تقييم المحتوى وفق 70 معيار EEAT (متكيف مع النضج)
4. التحقق من Google Business Profile
5. التحقق مما إذا كان ChatGPT / Perplexity يستشهدان بهذا الموقع بالفعل
6. المقارنة مع أفضل 3 منافسين

→ **تقرير كامل مع إجراءات ذات أولوية، تقدير الجهد، وكود جاهز للنسخ.**

## نظام التطور الذاتي

تتحسن المهارة مع الوقت **بدون إعدادات**:

1. بعد كل أمر، تُسجل النتائج في `evolution/learnings/`
2. تتراكم الأنماط المتكررة في `evolution/pattern-library.md`
3. تُتتبع تعديلات التقييم في `evolution/calibrations.md`
4. عندما يتجاوز العدد 50 ملفاً، تُؤرشف التعلمات القديمة وتُدمج
5. ملفات التعلم تُنشأ فقط عند وجود نتائج غير واضحة

ملفات المرجع لا تُعدل أبداً تلقائياً. المستخدم يقرر المعايرة.
